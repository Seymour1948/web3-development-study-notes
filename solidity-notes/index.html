Solidity Notes

msg

The msg-object is a global always-existing object containing a few informations about the ongoing transaction. The two most important properties are .value and .sender. Former contains the amount of Wei that was sent to the smart contract. Latter contains the address that called the Smart Contract.

view

a view function is a function that doesn't alter the storage (read-only) and can return information. It doesn't need to be mined and it is virtually free of charge.

balance

A variable of the type address always has a property called .balance which gives you the amount of ether stored on that address.

payable

Allowed to send eth. Since 0.8.1 msg.sender is not payable. payable(msg.sender) to allow send eth.

block.timestamp

Obtain current time in the smart contract. You can use that to establish lock times or warmups.

require(condition, message)

Control structure to revert transaction if condition is not satisfied.
