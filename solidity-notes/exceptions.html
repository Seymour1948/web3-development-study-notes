Exceptions. Require, Assert, Revert

Transactions are atomic

When transaction fails at some point none of the transaction happens and everything is reverted.

Highlevel functions or interactions cascade

Meaning that if something fails in and interaction width another smart contract the whole transactions also is reverted in the contract.

That not happend with low level functions: address.send, address.call, address.delegatecall, address.staticcall

Revert and requires allow to return an string message

Assert vs Require

They had different bytecodes, meaning EVM react to them differently

Assert checks internal states and to handle unexpected errors inside contract

Require used for input validation

Assert consume all gas of the transaction, require only cosume the expeded till then The remaining is returned.

Assert is triggered by

  • Out of bounds index
  • Division by zero or module zero (5/0 or 23%0)
  • Byteshifting by a negative amount
  • Convert a value too big or negative to enum
  • Zero-initialized varialbe of internal function type
  • ... Or assert(x) where 'x' evaluates to false

Require is triggered by

  • Require(x) where 'x' evaluates to false
  • function call via message call but it doesn't finish properly.
    • Except low-level functions calls
  • External function call to a contract not containing any code
  • Your contract receives ether without a payable modifier
  • Your contract receives ether at a getter function
  • Address.transfer() fails

Revert

Revert can be use instead of require.

if (msg.sender != owner) { revert("Condition not fullfilled") }
